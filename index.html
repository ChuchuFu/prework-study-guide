HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user
  • !DOCTYPE tells the browser what document type it should expect when it is opened
  • HTML attributes define additional information about an element
  • "lang" tells the browser the language for the webpage. "en" stands for English
  • HTML attributes are always defined in the beginning element, never the closing element
  • Metadata is data about our webpage that the browser uses to effectively serve the webpage to users
  • The title element defines the title for a webpage
  • The title element is required and is extremely important for Search Engine Optimization, or SEO
  • The style element defines the CSS styles associated with a website
  • The base element defines the base URL for a webpage
  • The link element connects an external resource to the HTML document
  • The meta element defines meta data such as the character set, description, keywords, author, and viewport
  • The script element embeds client-side scripts in the HTML document
  • The charset attribute inside the head element sets the character encoding
  • The http-equiv attribute inside the head element sets an HTTP header for content
  • The viewport attribute sets the default view for the webpage
  • Elements that are nested inside other elements are called child elements
  • The element that holds all the children elements is known as the parent element
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage
  • The non-semantic element div is used to create a division in the page
  • The non-semantic element span is used to create an inline division
  • We insert the content between an opening tag and a closing tage so that the browser understands how to treat the content
  • An element refers to the entire element from opening tag to closing tag
  • A tag refers only to what appears inside the angle brackets
  • The elements from h1 to h6 are the levels of headings a given text block represents
  • The p element is a paragraph or block of text
  • The ul element is used for unordered lists
  • The ol element is used for ordered lists
  • The li element is used for list items
  • In essence, the unordered lists are bulleted, and the ordered lists are numbered
  • The image element is used for images displayed on the webpage
  • The src attribute contains the location of image
  • The br element creates a line of empty space between two blocks of content
  • The "a" element stands for anchor, which creates links to the same webpage or other webpages
  • The href attribute points to the URL for the link
  • The header element appears at the top of the page, and usually contains a navigation menu and/or introductory content
  • The header tags need to be in order from h1 to h6
  • The element h1 should only appear once on each page
  • The alt attribute contains a text string that describes the appearance and functionality of the image
  • The footer element appears at the bottom of the page, and usually contains the author, copyright, contact, sitemap, and navigation
  • The section element defines a section in a document that contains thematic content
  • The id attribute is a way to identify elements and select them

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • pwd: stands for "print working directory", which displays the current working directory
  • ls: diplays the list of files and folders in the current directory
  • cd: navigates between directories in the paths
  • cd ..: navigates backwards, one level at a time, through the path of your directories
  • mkdir: creates a new directory
  • touch: creates a file inside a directory
  • start: opens a file in the Windows System
  • The "git" command must precede every Git statement
  • git status: checks which branch we are in
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add: adds modifications in the current working branch to the staging area
  • The "-A" command following the "git add" command is to add all changes
  • git commit: captures a snapshot of the currently staged changes
  • The key to writing effective commit messages is to write them as imperative command in the present tense
  • git pull: receives a branch's modifications into the local environment
  • git push: pushes the changes we have locally to remote Github branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
